# 1. 概述

Git是一个分布式版本控制工具，下载地址 (opens new window)

# 2. git常用命令

# 2.1 设置用户签名

设置用户名

git config --global user.name "用户名"

设置邮箱

git config --global user.email "邮箱"

# 2.1 简单命令

初始化仓库

git init

查看仓库状态

git status

添加文件至暂存区

git add <文件名>

添加所有文件至暂存区

git add .

将文件从暂存区中移除

git rm --cached <文件名>

提交本地库

git commit -m <日志信息>

查看简单日志信息

git reflog

查看详细日志信息

git log

版本穿梭

git reset --hard <版本号>

# 3. 分支操作

在版本控制过程中，同时推进多个任务，每一个任务我们都可以创建单独的分支。使用分支意味着程序员可以把自己的工作从开发主线上分离开来，开发自己分支的时候，不会影响主线的运行。使用分支可以并行推进多个功能开发，提高开发效率。各个分支在开发过程中 ，如果某一个分支开发失败，不会对其他分支有任何影响

# 3.1 常用操作

创建分支

git branch <分支名>

查看分支

git branch -v

切换分支

git checkout <分支名>

把指定的分支合并到当前分支上

git merge <分支名>

删除分支

git branch -d <分支名>

重命名分支

git branch -m <旧名字> <新名字>

# 3.2 冲突合并

冲突产生的原因，合并分支时，两个分支在同一个文件的同一个位置有两套完全不同的修改。Git无法替我们决定使用哪一个。必须人为的决定新代码的内容

# 4. Github

地址 (opens new window)

# 4.1 git常用远程命令

查看远程仓库

git remote -v

添加一个远程仓库

git remote add <别名> <仓库地址>

删除一个远程仓库

git remote rm <别名>

推送远程库

git push <别名> <分支>

拉取远程仓库

git pull <别名> <分支>

克隆仓库

git clone <仓库地址>

# 4.1 团队协作

  1. 邀请成员加入

  1. 复制邀请函

  2. 邀请函发给对方，对方同意即可

  3. 对方同意后，自己的仓库就会出现在对方的仓库列表中

# 4.2 跨团队协作

  1. 首先将对方的仓库fork到自己的账号下

  2. 修改代码

  3. 创建Pull requests

  4. 创建好后发给对方，等待对方审核即可

# 5. SSH登陆

  1. 生成密钥

    ssh-keygen.exe -t rsa

  2. 将C:\Users\<windows用户名>\.ssh\id_rsa.pub的内容复制到github即可

  1. 创建一个xxx.gitignore文件(前缀名是什么无所谓)
  2. 原则上忽略文件放在哪里都行，建议放在根目录
